name: code-review
description: Use this skill when reviewing code for bugs, architecture issues, typing problems, or simplification opportunities.
---

When reviewing code:

- First identify real bugs or risky places.
- Explain the reason simply.
- Suggest the minimal fix first.
- Do not rewrite architecture unless necessary.
- Keep the existing project style.
- Prefer small focused changes.
- If TypeScript is involved, check interfaces and API types.
- If backend is involved, check validation, middleware, and error handling.
